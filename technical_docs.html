01. ENGINE STATE & BOOT

window.onload
Initializes canvas, binds input, creates initial path, and prepares start-screen state. If save data exists, start screen offers loadGame().
resetGameLogic()
Hard runtime reset for a new run. Reinitializes economy, lives, wave counters, abilities, entities, and initial path graph.
saveGame() / loadGame() / queueAutoSave()
Serializes run state to localStorage.neonDefenseSave. Restores towers, rifts, economy, and progression metadata. If a saved run was mid-wave, load flow converts it to a short prep phase (prepTimer = 5). Audio settings are stored separately via localStorage.neonAudioSettings. queueAutoSave() debounces saves with a minimum frame interval to avoid per-frame writes; triggered on wave start, tower builds, upgrades, sells, and ability use. saveGame() snapshots all state synchronously then defers the expensive JSON.stringify + localStorage.setItem call to requestIdleCallback() (5 s deadline fallback to synchronous) so it never blocks a render frame.
startGame() / fullReset()
Entry and wipe flows. startGame() launches gameplay, fullReset() clears persistent save/tutorial keys and reloads the app.

02. FRAME LOOP

gameLoop(timestamp)
Main RAF callback. Runs update() while active and always runs draw().
update()
Advances wave timers, cooldowns, enemy/tower/projectile systems, particle simulation, and auto-save triggers on key progression events.
draw()
Full canvas redraw under camera transform, including paths, towers, enemies, VFX, and UI overlays.

03. WAVE ORCHESTRATION

startPrepPhase()
Sets pre-wave timer (30s), clears spawn queue, updates wave UI, and dispatches scheduled rift growth to the path worker via _requestRiftGeneration(). Generation runs off the main thread; each completed rift arrives as a path_ready message that applies side-effects (path push, tower destruction, UI update) on the main thread incrementally during the prep window.
startWave()
Builds the spawn queue and applies phase effects:
  • Base count: 5 + floor(wave * 2.5)
  • Guaranteed boss every 10th wave
  • After wave 50: optional surprise boss on non-10th multiples of 5 (25%)
  • Every 20th wave: random rift mutation assignment
  • After wave 50: each rift has 10% tier-up chance at wave start
spawnEnemy() / spawnSubUnits(parent)
Dequeues enemy type, selects a random rift, applies wave scale, rift tier multipliers, and mutation modifiers. Splitter death path routes to spawnSubUnits().

04. RIFT GENERATION & PATHING

calculatePath()
Generates the initial spawn path from a randomized start cell at least 10 grid units from center via findPathOnGrid(). Fallback path is applied if generation fails.
generateNewPath() / path_worker.js
Creates additional rifts using zone-based candidate search and merge behavior. For normal gameplay the full logic runs inside scripts/workers/path_worker.js off the main thread (see below). The main-thread generateNewPath() copy is kept for the debug debugCreateRift() hook only.
  • Zone ring bounds: inner 6 + (zone-1)*3, outer 6 + zone*3
  • Zone capacity based on orbital shell formula 2·zone²·0.62
  • Direct core-link probability: 0.5 / (zone^2)
  • Global spacing guard: wave-scaled minimum rift distance (0.65–0.95 grid units)
  • Merge candidates capped at 240 (420 aggressive); A* called up to 260 times per rift attempt
  • Three relaxed levels (0→2) tried automatically per attempt, then aggressivePlacement fallback
New path segments destroy towers in newly opened lanes and refund 70% of invested value (applied on the main thread after the worker posts a path_ready message).
_requestRiftGeneration() / _applyGeneratedRift() / _onPathWorkerMessage()
Path worker management layer in 03_abilities.js.
  • Message Main → Worker: { type: 'generate_batch', state: { cols, rows, wave, gridSize, zone0Radius, pathingRules, paths, hardpoints, count, maxAttempts } }
  • Message Worker → Main (per rift): { type: 'path_ready', newPathPoints, foundZone } — triggers _applyGeneratedRift() which pushes to paths[], destroys displaced towers, and calls updateUI().
  • Message Worker → Main (batch end): { type: 'batch_done', generated, remaining } — releases the busy flag; auto-continues if pendingRiftGenerations > 0 (handles the case where a new prep phase queued work while the worker was busy).
  • Worker is lazily created on first use; a single instance is reused across all waves.
  • maxAttempts formula: min(1800, 120 + count × 40).
findPathOnGrid(start, end, obstacles)
A*-style grid pathfinder with:
  • Obstacle crossing penalty: +100
  • Direction change penalty: +5
Uses Manhattan heuristic and reconstructs center-aligned world points.

05. COMBAT & ABILITIES

isValidPlacement(x, y, towerConfig)
Validates world-space tower placement against UI exclusion zones, player funds, path collisions, and occupied cells.
buildTower() / upgradeTower() / sellTower()
Tower economy and lifecycle entry points. Upgrade scaling and sell-refund logic are applied at runtime.
updateTowers() / shoot() / hitEnemy()
Targeting, projectile emission, and damage resolution. Kill handling updates money, total kill counters, energy gain, and conditional splitter spawn. Arc towers use instant chain-lightning resolution, static charge accumulation (ARC_TOWER_RULES.staticThreshold = 100), and 0.5s stun on threshold. Death particles spawn only on kill; arc hit particles were removed to reduce per-frame cost. Tower targeting uses ENEMY_SPATIAL_GRID for O(1) broad-phase queries instead of O(n) scans.
activateAbility(type) / useAbility(type, target)
Ability state machine for:
  • emp: AOE freeze + frozen damage vulnerability
  • overclock: temporary fire-rate buff on a selected tower
Applies energy costs and cooldown tracking in seconds.

06. USER INTERFACE

setupInput() / screenToWorld() / resetCamera()
Mouse/touch abstraction for panning, zooming, tap-select, and click-build workflows. Zoom range: 0.1× (min) to 1.0× (max). 1.0× is the default and matches the resetCamera() centered view. Both wheel and pinch handlers clamp to the same range. screenToWorld(x, y) converts screen-space coordinates to world-space using the current camera.x / camera.y / camera.zoom state.
updateUI()
Synchronizes HUD counters, wave timers, enemy composition badges, energy bar, and cooldown states.
updateSelectionUI()
Renders context panels for selected tower, core, or rift intel (tier and mutation modifiers).
updateWavePanel()
Builds wave-intel content from current rift graph and incoming wave metadata. Core fields (count/anomaly/threat) always update; extended special-list output requires a matching panel target id in the DOM.
getRepairCost()
Dynamic repair-price helper used by core panel controls.
updatePerformanceUI() / adjustDetails(delta) / showQualityToast(msg)
Quality stepper UI bindings exposed on window.
  • updatePerformanceUI(): syncs the GAME DETAILS label and −/+ button disabled states to QUALITY_GOVERNOR.profileIndex (actual running level, not floor).
  • adjustDetails(delta): delta = +1 improves quality (index−1), delta = -1 reduces (index+1). Updates floor, persists to localStorage, and calls refreshQualitySettings().
  • showQualityToast(msg): displays a fade-in/out overlay at top-right (below top bar) for 2.2s. Timer is reset on rapid successive calls.

07. AUDIO ENGINE

AudioEngine.init()
Creates Web Audio graph (master/music/sfx gains) and resumes suspended contexts on demand.
AudioEngine.updateMusic()
16-step sequencer with 15 normal melody variants and one threat profile. Automatically switches based on active boss/mutant presence and wave index.
AudioEngine.setVolume() / toggleMute()
Applies gain changes and persists music/SFX/mute settings to local storage.
AudioEngine.playSFX(type)
Procedural synth SFX helper used by tower fire, explosions, hits, and system feedback events.

08. PERFORMANCE SYSTEMS

QUALITY_GOVERNOR / QUALITY_PROFILES / applyQualityProfile()
Three quality presets (indexed 0–2) control VFX budgets per frame.
  • HIGH (0): 900 particles · 140 lights · 180 arc bursts · arc links (intensity-styled) · full alpha · all effects update every frame.
  • MED (1): 620 particles · 90 lights · 140 arc bursts · arc links (intensity-styled) · chain bursts update every 2 frames.
  • LOW (2): 420 particles · 65 lights · 96 arc bursts · arc links (intensity-styled) · chain bursts every 3 frames.
Arc link visuals are not quality-gated; all three presets render the same intensity-based dot → dash → solid-glow progression (see Arc link batching below). QUALITY_GOVERNOR.profileIndex is the active level. The governor auto-downgrades when frameDt > 22ms for 45 consecutive frames, and auto-upgrades when frameDt < 15.8ms for 240 frames. PERFORMANCE_RULES.qualityFloor is the user-set minimum (0/1/2); the governor cannot go above qualityFloor in index terms. PERFORMANCE_RULES.autoDropEnabled (persisted in localStorage) gates the entire downgrade/upgrade logic — when false the governor freezes at the current preset. ARC_TOWER_RULES.lowAnimationMode is set to true whenever qualityFloor > 0; it affects chain-burst stride but not arc link visuals.
ENEMY_SPATIAL_GRID / queryEnemiesInRadius() / queryTauntersInRadius()
Spatial hash grid with cellSize = 200 world units. Rebuilt each frame from live enemies. Replaces O(n) distance scans for:
  • Tower targeting (nearest enemy / nearest taunter in range)
  • Arc chain-bounce candidate selection
  • Base turret targeting
Query results are written into caller-supplied scratch arrays to avoid per-call allocations. Squared distance comparison is used throughout (no Math.hypot in hot paths).
Render Pipeline — Batching and Caching
Key render optimizations in 06_render.js:
  • Enemy multi-pass batching: Pass 0 classifies into color buckets and a special list. Pass 1 draws one beginPath/fill per color (shadow set once). Pass 2 draws healers and invisible shifters individually. Pass 3 batches elite diamond markers. Pass 4 draws health bars in two sweeps (red then green); full-HP enemies are skipped entirely.
  • Particle alpha quantization: 8 discrete alpha levels; one globalAlpha + beginPath/fill per (level, color) group per frame.
  • Arc link batching: links are pre-bucketed by intensity level (1–5) in a single O(N) pass into _linkBuckets[5]. Visual style is driven by the _LINK_STYLES constant (allocated once at module load, never per-frame):
    • Levels 1–4: lineCap='round' stroke with increasing dash segment lengths ([1,16] → [2,11] → [5,8] → [12,4]). The round cap on a 1 px segment renders as a circle dot; longer segments produce pills then dashes — a continuous visual progression from a single style idiom.
    • Level 5: double-stroke on the same path (no second beginPath()): a wide low-alpha halo stroke (10 px, rgba(120,210,255,0.28)) followed by a narrow high-alpha core stroke (2.4 px, rgba(200,245,255,0.94)). No shadowBlur; no quality-mode gate.
    Maximum 5 stroke() calls per frame for levels 1–4; 2 for level 5 (same path, stroked twice). Total max 6 stroke calls regardless of link count.
  • Arc segment cache: _arcSegCache map stores typed arrays (Float32Array / Int8Array) of pre-computed t, envelope, zig, sinArg, and cosArg per segment count. Only Math.sin/cos per segment per frame remain in the hot path.
  • Light gradient cache: radial gradients are pre-baked to offscreen canvases keyed by (color, radius) and drawn with drawImage each frame. No createRadialGradient() calls at runtime.
  • Boss hexagon lookup: _HEX_COS / _HEX_SIN Float32Array pre-baked from Math.cos/sin(i * Math.PI / 3) — zero trig calls per boss per frame.
  • globalCompositeOperation: 'screen' eliminated: removed from arc links, arc bursts, and dynamic lighting. All use source-over with tuned alpha to avoid GPU framebuffer read-back.
Async Rift Generation — Web Worker (C4-17)
scripts/workers/path_worker.js is a self-contained Web Worker that runs the full A* rift-generation pipeline off the main thread.
  • Contains complete copies of tryGenerateAtLevel(), findPathOnGrid(), and all 8 pure-math helpers (isGridNearHardpoint, getCoreRepulsionPenalty, getCoreGapSectors, etc.) — no DOM, no canvas, no shared state.
  • Batch loop mirrors original failStreak thresholds (rl escalates at 3/9 failures; aggressive placement at 14); main thread only applies side-effects.
  • Eliminates the worst-case 50–2000 ms synchronous stall during prep phase and game-reset topology rebuilds.
Deferred Save — requestIdleCallback (C4-18)
saveGame() captures a snapshot of all game state synchronously (cheap), then defers JSON.stringify + localStorage.setItem to requestIdleCallback() with a 5 s deadline. On browsers without requestIdleCallback the write is synchronous as before. Removes 5–20 ms serialization spikes from the render frame on late-game saves with large rift/tower payloads.
